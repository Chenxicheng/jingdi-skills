# 反馈与状态

## 反馈组件选择

- `Alert`：用于页面或区块内需要持续关注的说明、警告和错误。
- `Notification`：用于跨页面、后台任务或需要稍后回看的重要结果。
- `Badge`：用于未读数量、待处理数量或轻量状态；数量规则以 project 为准。
- `Popover`：用于当前上下文中的补充信息或轻量操作，保留清晰关闭路径。
- `Tooltip`：只承载短说明，关键任务、复杂内容和必须发现的信息保持可见。
- `Message`：用于短暂、低风险且结果无需持续保留的反馈；结果已直接显示时省略重复提示。
- `Dialog`：用于阻塞决策、高风险确认或必须在继续前处理的内容。

## 加载与进度

- 已知页面骨架使用 `Skeleton`，小范围未知等待使用 `Spinner`，可测量长任务使用 `Progress`。
- 长任务说明当前阶段和预期结果；能取消时提供取消，适合转后台时允许用户离开。
- 加载期间保持页面结构稳定，避免按钮、列宽和内容频繁跳动。
- 防止重复提交，同时保持无关区域可用。

## 状态完整性

把正常、加载、空、错误、无权限、禁用、提交中、成功和数据过期视为同一功能的组成部分。

- 空状态区分首次无数据、筛选无结果、主动清空和无权限，并提供对应下一步。
- 错误状态说明影响范围，提供重试、返回、查看详情或联系支持等可行动路径。
- 权限受限但用户有知情权时解释原因和申请方式；用户无知情权时隐藏入口和对象信息。
- 并发修改、数据过期或权限变化时说明冲突，保留输入并提供刷新、覆盖或另存路径。
- 成功反馈帮助用户确认结果和继续下一步，同时保留当前上下文。
