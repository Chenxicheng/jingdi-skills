# 页面模式

## 列表页

- 按“标题与主操作、查询与筛选、批量操作、数据区、分页”组织页面。
- 区分 Search、Filter 和 Query：关键词搜索对象，多选条件即时筛选，复杂条件统一查询。
- 常用条件持续可见，其余进入“更多筛选”；持续显示已生效条件并允许一键清除。
- 行级操作只保留高频项，其余进入更多菜单；选择数据后再显示批量操作。
- 新建成功后保留筛选与排序上下文，并突出新对象；删除后更新列表并说明结果或撤销方式。
- 区分首次无数据、筛选无结果和加载失败，分别提供开始使用、清除条件和重试。

## 详情页

- 先展示对象标识、状态、关键属性和主要操作，再按主题分组展示详情。
- 内容少时平铺；独立主题使用分区或卡片；真正并列且信息量大的主题使用 Tabs。
- 高频轻量编辑优先行内完成；多字段、复杂校验或需要独立链接的编辑进入 Drawer 或页面。
- 重要变更展示操作者、时间、前后值和来源，保证结果可追溯。
- 从列表进入详情后，返回时恢复原有查询和位置。

## 表单页

- 默认采用单列和从上到下的任务顺序，只有短小且强关联的字段才并排。
- 短表单一次完成；长表单按业务阶段分组并保存进度。
- 步骤条只用于真实线性依赖，每一步说明目的、完成条件和返回影响。
- 离开未保存页面时提供保存、放弃或继续编辑的明确选择。
- 成功后说明保存对象和下一步，同时保留任务上下文。

## 结果页

- 用于长流程结束、强结果反馈或需要保留大量结果信息的场景；轻量结果留在原页面反馈。
- 标题描述对象、动作和结果，正文补充关键数据、影响范围和有限的下一步。
- 清晰区分成功、部分成功和失败；批量任务展示成功与失败明细及重试路径。

## 异常页

- 明确异常类型和影响范围，提供重试、返回、联系支持或查看详情。
- 无权限、资源不存在、服务不可用和网络异常使用不同文案与恢复路径。
- 保留用户正在处理的对象、输入和导航上下文，并提供可继续路径。

## 工作台

- 围绕角色回到系统后的首要任务组织内容。
- 首屏优先放待办、异常、常用入口和关键业务状态，低频统计与帮助后置。
- 根据权限和职责调整内容，同时保持模块位置与命名稳定。
- 每个模块提供处理下一项、查看全部或返回来源页面的路径。
- 工作台只保留具有明确行动价值的任务、状态和统计。

## 可视化页面

- 先总览，再筛选，再按需查看明细；筛选条件和统计口径对全部图表保持一致。
- 一张卡片表达一个主题，关键指标置前，辅助解释与异常原因就近展示。
- 联动筛选时明确影响范围，并提供取消选择和恢复全局视图。
- 数据延迟、缺失、异常和口径变化保持可见。

## 空状态

- 说明为什么为空，以及用户现在能做什么。
- 首次使用引导创建或导入；筛选无结果提供清除条件；主动清空确认结果；无权限提供合适申请路径。
- 插图辅助情绪和识别，标题、原因和操作负责传达关键信息。

## 消息与反馈模式

- 就地变化优先就地反馈；短暂低风险结果使用 Message；持续问题使用 Alert。
- 跨页面或后台任务进入 Notification 或消息中心，并保留可回看记录。
- 高风险阻塞决策使用 Dialog，内容明确对象、后果、主次动作与关闭方式。
- 一次操作采用足以确认结果的最小反馈组合。
