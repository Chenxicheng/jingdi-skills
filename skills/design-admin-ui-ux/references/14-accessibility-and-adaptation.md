# 可访问性与适配

- 以 WCAG 2.2 AA 为基线，并遵循 project 更严格的要求。
- 保证键盘可达、焦点顺序合理、焦点样式清晰；浮层打开后管理焦点，关闭后返回触发点。
- 为输入提供可关联的 Label、说明和错误，为图标按钮、状态和图表提供可访问名称或替代表达。
- 优先使用语义结构和原生控件；自定义交互补齐角色、状态和键盘行为。
- 文字与非文字内容满足对比度要求，并用文字、图标或形状补充颜色语义。
- 支持文本缩放、长文本、多语言、窄窗口、屏幕阅读器和减少动态效果偏好。
- 触控目标与目标间距遵循 WCAG 和 project 约定，高频操作提供足够热点。
- 响应式设计定义内容如何伸缩、折行、折叠和滚动。
- 窄窗口优先保留主任务、主操作和关键数据；次要内容进入展开区、Drawer 或详情页。
- 验证正常、加载、空、错误、无权限、禁用、提交中和成功状态下的缩放、键盘与焦点行为。
