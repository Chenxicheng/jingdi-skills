# 导航

- 导航回答“我在哪里、能去哪里、如何返回”。清晰显示当前模块、页面和对象位置。
- 模块少、层级浅且切换频繁时可使用顶部导航；模块多、层级深或操作密集时优先侧边导航。
- 按业务任务和对象组织菜单。菜单、页面标题和操作文案使用同一术语。
- 面包屑只表达真实层级和返回关系。
- Tabs 只承载同一对象或任务下的并列内容；内容存在先后依赖时使用步骤，存在上下级关系时使用导航。
- 侧边导航折叠后仍要保留当前位置、图标含义和可访问名称。
- 返回列表时恢复筛选、排序、分页、滚动位置和选中状态。
- 分页、无限滚动和虚拟列表按数据规模与任务选择；需要定位、比较和返回时优先保留稳定页码或位置。
