# 使用方式与决策优先级

本指引面向管理后台、运营平台、控制台、内部工具和数据密集型任务系统。开始设计或实现前，先读取 project 中的设计系统、design token、主题配置、组件封装、同类页面、业务术语、权限模型与响应式约定。

具体尺寸、颜色、字号、间距、圆角、阴影和动效参数均以 project 为准；本指引只规定选择逻辑和体验底线。

发生冲突时，按以下顺序裁决：

1. 用户能否完成任务，结果是否正确
2. 安全、权限与操作是否可恢复
3. 可访问性是否达标
4. 是否与 project 保持一致
5. 信息是否清晰
6. 操作是否高效
7. 视觉是否美观

每次设计都回答四个问题：谁在什么场景使用、核心任务是什么、成功结果是什么、失败后如何继续。以任务完成为目标组织界面。
