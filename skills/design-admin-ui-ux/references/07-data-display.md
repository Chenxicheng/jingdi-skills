# 数据展示

## 表格

- 表格用于扫描、比较和操作结构化数据；信息不能形成稳定列关系时考虑列表或卡片。
- 优先展示识别对象、判断状态和完成高频操作所需的列，其余信息进入详情或列设置。
- 文本左对齐，数值右对齐，状态和操作保持稳定位置；表头说明单位和统计口径。
- 支持排序、筛选或分组时明确当前条件，并提供恢复默认值的方式。
- 窄窗口先调整列优先级、固定对象标识和操作，再允许横向滚动。
- 为超长文本、极大数值、缺失值、未知值、无权限值和异常状态定义稳定展示。
- 批量操作只在选择数据后出现，并持续显示选择数量、作用范围和退出方式。

## 其他数据容器

- `Collapse` 用于按需展开次要详情，主任务的必需信息保持可见。
- `Card` 用于独立对象、主题或可整体操作的模块。
- `Tree` 用于真实层级和父子关系；深层结构提供搜索、展开策略和当前位置。
- `Timeline` 用于按时间追溯事件，区分操作者、时间、状态和来源。
- `Carousel` 仅用于内容平级且顺序不重要的浏览场景，中后台核心数据和操作通常不适用。

## 数据可视化

- 先定义用户需要作出的判断，再选择指标和图表。
- 趋势优先折线或面积图，分类比较优先柱状图，分布优先散点或箱线图，流程转化可用漏斗图。
- 占比图只在类别少、总和明确且比较目标简单时使用；精确比较优先条形图或表格。
- 每张图说明标题、口径、单位、时间范围、数据来源和更新时间。
- Tooltip 提供准确值；图表表达趋势和关系，表格表达精确值与明细。
- 关键图表提供文本摘要或可访问表格，让颜色、形状和悬停之外仍可读取数据。
