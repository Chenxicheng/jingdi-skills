# 数据录入

## 控件选择

| 场景 | 优先选择 | 关键约束 |
| --- | --- | --- |
| 少量、可同时比较的互斥选项 | `Radio` | 展示全部选项和关键差异 |
| 选项较多或需要搜索 | `Select` / `Combobox` | 支持搜索、清空、加载与无结果 |
| 修改后立即生效的二元状态 | `Switch` | 标签描述设置本身，切换后立即反馈 |
| 多项选择后统一提交 | `Checkbox` | 保留明确提交动作与已选状态 |
| 精确数值 | `InputNumber` | 显示单位、范围、步长与错误原因 |
| 范围或强度探索 | `Slider` + `InputNumber` | 同时支持快速调整与精确输入 |
| 文件或批量数据 | `Upload` | 说明格式、大小、数量、进度与失败恢复 |

## 字段表达

- Label 说明“填什么”，帮助文本说明“为什么或如何填”，Placeholder 只用于格式示例。
- 必填、选填、单位、格式和限制在输入前可知。
- 提供合理默认值和上下文带入，同时保留重要决策的显式确认。
- 关联字段放在一起；字段存在依赖时及时更新可用状态和说明。
- 输入较长或成本较高时自动保存草稿，或在离开前提供保存、放弃和继续编辑。

## 校验与提交

- 在字段附近显示具体、可修复的错误，并保留正确输入。
- 输入中只提示能即时确定的问题；需要服务端判断的错误在适当时机校验。
- 提交失败时聚焦或定位首个错误，并在页面级说明整体结果。
- 提交中阻止重复动作，保持按钮宽度和当前内容；完成后明确保存结果。
- 设置项无依赖时可以即时生效；相互依赖时统一提交，同一页面采用一致保存模型。
