# 设计价值观

## 为目标设计

- 按用户任务组织页面，不按数据库结构、内部部门或技术模块堆叠功能。
- 先建立一条可直接完成的主路径，再补充取消、返回、失败和恢复路径。
- 删除不帮助当前任务的模块、字段、入口和说明。

## 让系统承担复杂性

- 能从上下文推断的内容不要求用户重复填写。
- 能自动保存、自动带入或给出合理默认值的内容，不让用户从空白开始。
- 能一步完成的任务不拆成多步；只有真实依赖才使用向导或步骤条。

## 保持一致与可预测

- 复用 project 的组件、token、术语和同类交互，不为局部页面创造同义模式。
- 相同操作保持相同位置、文案、反馈和风险等级。
- 一致性与任务正确性冲突时，修正规则或记录缺口，不复制明显错误。

## 渐进式展示

- 首屏呈现任务必需的信息和操作，细节按需展开。
- 高频操作持续可见，低频操作进入更多菜单、展开区或详情页。
- 关键规则、风险和状态默认可见。

## 反馈引导行动

- 反馈说明发生了什么、影响什么、下一步能做什么。
- 反馈尽量出现在操作或问题发生处；全局反馈只用于跨页面或后台事件。
- 保留用户输入和操作上下文，避免失败后从头开始。
