# 视觉基础

## 色彩

- 只使用 project 的语义 design token、主题配置和数据可视化 token，不在局部实现中另定义色号。
- 颜色用于表达品牌、状态、层级和交互反馈，同时提供文字、图标、形状或位置线索。
- 图表中同一指标、类别和状态保持稳定映射；需要比较的数据使用易区分的颜色。
- 暗色与高对比主题下重新验证层级、对比度和状态含义。
- project 缺少所需语义 token 时，先补齐 token，再在业务页面使用。

## 布局与间距

- 使用 project 的栅格、容器和间距尺度。
- 通过间距表达关系：同组元素靠近，跨组元素拉开，重要层级获得稳定留白。
- 页面建立清晰视觉起点；标题、筛选、正文、表单和表格遵循可预测的对齐线。
- 卡片只用于独立主题、需要边界或可整体操作的模块；普通分组优先使用标题、留白和分割。
- 高频专业工作允许紧凑，同时保证辨识、点击准确性与错误恢复；低频或高风险任务提供更多说明和间距。
- 页面内容过多时，先按任务优先级删减、分组或下钻。

## 字体排版

- 使用 project 的字体、字号、字重、行高和文本 token。
- 页面标题、区块标题、正文、辅助文字和数据建立有限且稳定的层级。
- 正文以易读为先；辅助文字降低视觉权重时仍需满足对比度和缩放要求。
- 数值列右对齐，比较型数据统一精度与单位；需要纵向比较时使用等宽数字能力。
- 长标题、长名称和多语言文本允许换行、截断后查看全文，或使用更合适的容器。
- 使用有限强调，维持稳定阅读层级。

## 图标

- 复用 project 图标库，保持线条、填充、透视和尺寸风格一致。
- 图标用于帮助识别；陌生、关键或高风险操作同时显示文字。
- 只有图标的按钮必须提供可访问名称；含义可能不明确时提供 Tooltip。
- 同一区域使用易区分的操作与状态图标。
- 自定义图标遵循 project 的画布、描边和对齐规则，并验证小尺寸识别性。

## 阴影与层级

- 阴影只表达真实的空间层级。
- 下拉、Popover、悬浮工具栏、Drawer 和 Dialog 按 project 的浮层层级递增。
- 同层浮层使用一致阴影；层级由位置、遮罩、边界和交互状态共同表达。
- 内容卡片默认依赖间距、背景和边框分组，仅在确有悬浮含义时使用阴影。

## 暗色模式

- 暗色模式复用与明色模式相同的语义 token 体系和信息结构。
- 分别校验文字、边界、禁用态、状态色、图表、插图和浮层。
- 控制大面积高亮与强对比，同时保持焦点、选中和错误状态清晰。
- 用户切换主题后保持任务上下文和未保存内容。
