# 文案与数据格式

- 从用户视角描述任务、对象和结果，使用 project 的统一业务术语。
- 标题说明当前对象或任务，按钮说明动作结果，帮助文本补充原因、规则和示例。
- 错误文案采用“发生了什么 + 影响什么 + 下一步”的结构，同时保留必要错误标识。
- 删除、停用、覆盖等高风险动作直接写明对象和后果。
- 日期、时间、时区、数字、金额、百分比、计量单位与缺失值遵循 project 的 locale 和格式规则。
- 需要准确追溯时显示绝对时间；强调新近程度时可以显示相对时间，并允许查看准确值。
- 比较数据保持相同精度、单位和统计口径；缩写必须保留正确量级。
- 缺失、未知、未加载、无权限和不适用使用不同表达。
- 敏感信息按 project 的安全策略脱敏，说明查看权限和数据用途，提示中只显示完成任务所需的数据。
